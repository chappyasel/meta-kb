# Awesome Autoresearch

> A curated, high-signal index of autonomous improvement loops, research agents, and descendants inspired by Karpathy's autoresearch -- serving as the definitive map of the self-improving agent ecosystem.

## What It Does

Awesome Autoresearch catalogs every notable project in the autoresearch lineage: general-purpose descendants (Claude Code skills, Codex ports, Gemini adaptations), research-agent systems (end-to-end scientific pipelines), platform ports (macOS, Windows RTX, WebGPU, Jetson), domain-specific adaptations (trading, kernel optimization, genealogy, sudoku solving), and evaluation benchmarks. It also curates writeups and case studies from notable applications like Shopify's Liquid engine optimization and Vesuvius Challenge ink detection.

## Architecture

The repo is a standard awesome-list: a single README organized into categorized sections with GitHub star badges for social proof. Each entry includes a description of what makes it distinct from other autoresearch variants. Categories cover:

- **General-purpose descendants**: pi-autoresearch, codex-autoresearch, gemini-autoresearch, GOAL.md pattern, ADAS, SICA, GEPA
- **Research-agent systems**: AI Scientist, NanoResearch, AgentLaboratory, AutoResearchClaw
- **Platform ports**: macOS/MLX, Windows RTX, WebGPU, Jetson AGX Orin, Colab/Kaggle T4
- **Domain-specific**: Trading agents, GPU kernels, biomechanics, sudoku solvers
- **Evaluation benchmarks**: MLAgentBench, mle-bench, AgentBench

## Key Numbers

- **1,169 GitHub stars**, 94 forks
- **50+ cataloged projects** across 6 categories
- Licensed CC0-1.0 (public domain)

## Strengths

- The categorization reveals that autoresearch loops are becoming composable primitives -- not a single tool but a pattern applied across domains, platforms, and evaluation harnesses
- Curated notable use cases with verifiable traces (Shopify PR, Driveline biomechanics, Vesuvius Challenge) provide evidence that the pattern works at production scale

## Limitations

- As a list repo, it has no executable code -- it is an index, not an implementation
- Rapid ecosystem growth means entries can go stale; star counts and activity levels vary widely across listed projects
