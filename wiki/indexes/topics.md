# Topics Index

- [Retrieval-Augmented Generation](../concepts/retrieval-augmented-generation.md) — A technique combining retrieval systems with generative models to ground outputs in external knowledge. Includes agentic variants where agents dynamically retrieve information during reasoning.
- [Episodic Memory](../concepts/episodic-memory.md) — A type of agent memory that stores specific past events, interactions, and experiences, enabling agents to recall and learn from prior episodes.
- [Context Engineering](../concepts/context-engineering.md) — The discipline of designing, structuring, and optimizing the information provided to LLMs in their context window, encompassing prompt engineering, prompt architecture, and dynamic context construction.
- [Model Context Protocol](../concepts/model-context-protocol.md) — Anthropic's open protocol for standardizing how LLM applications communicate with external tools and data sources, enabling interoperable MCP servers.
- [Knowledge Graph](../concepts/knowledge-graph.md) — A structured representation of entities and their relationships used as a knowledge substrate for agents, enabling complex reasoning over interconnected information.
- [ReAct](../concepts/react.md) — Reasoning and Acting framework for LLM agents that interleaves chain-of-thought reasoning with action execution, enabling more reliable task completion.
- [Semantic Memory](../concepts/semantic-memory.md) — A type of agent memory storing general world knowledge, facts, and concepts, as distinct from episodic or procedural memory.
- [Agent Memory](../concepts/agent-memory.md) — The general capability of AI agents to store, retrieve, and utilize information across interactions, encompassing episodic, semantic, procedural, and core memory types.
- [Agent Skills](../concepts/agent-skills.md) — Reusable, composable capabilities that agents can acquire and invoke, including skill libraries, registries, manifests, and composition mechanisms for building capable agent systems.
- [Progressive Disclosure](../concepts/progressive-disclosure.md) — A context engineering strategy that reveals information incrementally based on agent needs, reducing context bloat while maintaining access to relevant details.
- [Long-Term Memory](../concepts/long-term-memory.md) — Agent memory that persists across sessions and conversations, enabling continuity of knowledge and personalization over extended time horizons.
- [Reflexion](../concepts/reflexion.md) — A framework for agent self-improvement through verbal reinforcement, where agents reflect on past failures and generate improved strategies stored in memory.
- [GEPA](../concepts/gepa.md) — Generative Evolutionary Prompt Architecture - a self-improving approach that uses evolutionary algorithms to optimize agent prompts and architectures.
- [CLAUDE.md](../concepts/claude-md.md) — A convention for storing persistent agent instructions, project context, and skill specifications in markdown files that are automatically included in agent context.
- [GRPO](../concepts/grpo.md) — Group Relative Policy Optimization, a reinforcement learning algorithm used to fine-tune LLMs through group-based reward comparison without a separate critic model.
- [Semantic Search](../concepts/semantic-search.md) — Search techniques that use embedding-based similarity to retrieve semantically relevant content, including vector search and vector store implementations.
- [Context Management](../concepts/context-management.md) — Strategies and systems for managing LLM context windows, including compression, pollution prevention, fragmentation handling, and lossless management approaches.
- [Procedural Memory](../concepts/procedural-memory.md) — A type of agent memory encoding how-to knowledge, workflows, and procedures that agents can execute, analogous to implicit skill memory in humans.
- [Multi-Agent Systems](../concepts/multi-agent-systems.md) — Systems involving multiple AI agents that collaborate, coordinate, or compete to accomplish tasks, including workflows, orchestration, swarms, and subagent hierarchies.
- [Vector Database](../concepts/vector-database.md) — Specialized databases optimized for storing and querying high-dimensional embeddings, including systems like ChromaDB, Pinecone, Qdrant, FAISS, and pgvector.
- [BM25](../concepts/bm25.md) — A classical sparse retrieval algorithm used in keyword-based search, often combined with semantic search in hybrid retrieval systems for agents.
- [Human-in-the-Loop](../concepts/human-in-the-loop.md) — An approach where human oversight, feedback, or approval is integrated into agent workflows at critical decision points.
- [Continual Learning](../concepts/continual-learning.md) — The ability of agents to continuously learn and adapt from new experiences without catastrophically forgetting previously acquired knowledge.
- [Execution Traces](../concepts/execution-traces.md) — Records of agent actions, decisions, and intermediate states during task execution, used for debugging, evaluation, and trace-derived feedback for improvement.
- [Lost in the Middle](../concepts/lost-in-the-middle.md) — A phenomenon where LLMs fail to effectively use information placed in the middle of long contexts, motivating research into context organization strategies.
- [Self-Improving Agents](../concepts/self-improving-agents.md) — AI agents that can modify their own behavior, skills, or architecture based on experience, including self-improvement loops and open-ended evolution approaches.
- [Core Memory](../concepts/core-memory.md) — A persistent, always-in-context memory block for agents containing essential information about the agent and user, shared across agent instances.
- [Community Detection](../concepts/community-detection.md) — Graph algorithms for identifying clusters of related entities in knowledge graphs, including the Leiden algorithm used in GraphRAG for community summarization.
- [Context Graphs](../concepts/context-graphs.md) — Graph-based representations of contextual information and agent computation flows, enabling structured context management and agentic computation graphs.
- [Context Compression](../concepts/context-compression.md) — Techniques for reducing context window usage while preserving essential information, including prompt compression methods like LLMLingua and LongLLMLingua.
- [Cognitive Architecture](../concepts/cognitive-architecture.md) — The structural organization of an AI agent's cognitive components including perception, memory, reasoning, and action, inspired by cognitive science models.
- [Hybrid Search](../concepts/hybrid-search.md) — A retrieval approach combining dense semantic search with sparse keyword search, often using reciprocal rank fusion to merge results for better coverage.
- [Temporal Reasoning](../concepts/temporal-reasoning.md) — The ability of agents to reason about time-ordered events, temporal relationships, and the evolution of knowledge over time in memory systems.
- [Reinforcement Learning](../concepts/reinforcement-learning.md) — A machine learning paradigm where agents learn through reward signals, applied to agent self-improvement including GRPO, reward hacking mitigation, and credit assignment.
- [SkillBook](../concepts/skill-book.md) — A structured registry or book of agent skills and their specifications, including SkillManager systems for organizing and accessing agent capabilities.
- [Chain-of-Thought](../concepts/chain-of-thought.md) — A prompting technique that elicits step-by-step reasoning from LLMs, including tool-integrated reasoning variants where reasoning is interleaved with tool calls.
- [Synthetic Data Generation](../concepts/synthetic-data-generation.md) — Techniques for creating artificial training data for agent improvement, including frontier-to-local distillation where larger models generate data for smaller ones.
- [Zettelkasten](../concepts/zettelkasten.md) — A note-taking and knowledge management methodology using atomic linked notes, adapted for agent knowledge bases through markdown wikis and wikilinks.
- [Prompt Optimization](../concepts/prompt-optimization.md) — Automated techniques for improving prompt quality and agent harness configurations, including meta-prompt optimization and automated harness optimization.
- [Organizational Memory](../concepts/organizational-memory.md) — Knowledge and information that persists within organizations or agent systems beyond individual interactions, including institutional memory and organizational judgment.
- [Case-Based Reasoning](../concepts/case-based-reasoning.md) — A problem-solving approach that uses experiences from similar past cases to inform new decisions, applicable to agent memory and retrieval systems.
- [Workflow Induction](../concepts/workflow-induction.md) — The process of automatically inferring or learning effective agent workflows from demonstrations, traces, or experience.
- [Incremental Indexing](../concepts/incremental-indexing.md) — A knowledge base maintenance strategy that updates indexes progressively as new information arrives, avoiding full reindexing overhead.
- [Short-Term Memory](../concepts/short-term-memory.md) — In-context memory available during a single agent session, including mechanisms for context retrieval and context reuse within the active window.
- [LLM-as-Judge](../concepts/llm-as-judge.md) — Using LLMs as automated evaluators to score or rank agent outputs, enabling scalable evaluation functions for self-improvement pipelines.
- [Personalized PageRank](../concepts/personalized-pagerank.md) — A graph algorithm variant used in knowledge graph retrieval to find relevant nodes by propagating relevance scores from query entities.
- [Entity Extraction](../concepts/entity-extraction.md) — NLP technique for identifying and extracting named entities and relationships from text, foundational to knowledge graph construction for agent memory.
- [Agent Harness](../concepts/agent-harness.md) — The scaffolding and evaluation infrastructure surrounding an agent, including meta-harness configurations, model harnesses, and agentic evaluation frameworks.
- [Docker Sandbox](../concepts/docker-sandbox.md) — Containerized execution environments providing safe, isolated sandboxes for agents to run code and system commands.
- [Observability](../concepts/observability.md) — Systems for monitoring, tracing, and debugging agent behavior in production, including tools like LangSmith and error logging infrastructure.
- [Memory Evolution](../concepts/memory-evolution.md) — The process by which agent memory systems adapt, reorganize, and improve over time through experience and automated optimization.
- [World Model](../concepts/world-model.md) — An agent's internal representation of the environment, entities, and their relationships, used for planning and decision-making.
- [Compositional Skill Synthesis](../concepts/composable-skills.md) — An approach to building agent capabilities by composing and synthesizing skills from simpler primitives, including automatic curriculum generation for skill acquisition.
- [Markdown Wiki](../concepts/markdown-wiki.md) — A knowledge base format using interconnected markdown files as a lightweight, human-readable knowledge substrate for agents and teams.
- [System Prompt](../concepts/system-prompt.md) — The initial instruction context provided to LLMs that defines agent behavior, persona, and constraints before user interaction begins.
- [Meta-Agent](../concepts/meta-agent.md) — A higher-level agent that oversees, coordinates, or improves other agents, taking on roles such as architect, curator, or reflector in multi-agent systems.
- [Tool Registry](../concepts/tool-registry.md) — A catalog of available tools and their specifications that agents can query and use, enabling dynamic tool discovery and selection.
- [Structured Output](../concepts/structured-output.md) — Techniques for constraining LLM outputs to follow specific schemas or formats, enabling reliable parsing and downstream processing in agent pipelines.
- [Context Generation](../concepts/abductive-context.md) — The process of dynamically generating relevant context for agents through autocontext and acontext systems that identify and surface pertinent information.
- [Agent-Mediated Workflows](../concepts/agent-mediated-workflows.md) — Business and technical processes orchestrated by AI agents, replacing or augmenting traditional automated workflows with adaptive agentic execution.
- [Compound Engineering](../concepts/compound-engineering.md) — An engineering approach that builds AI systems as compound products combining multiple models, tools, and services rather than relying on a single model.
- [Bounded Context](../concepts/bounded-context.md) — A domain-driven design principle applied to agent context management, limiting context scope to relevant domain boundaries for efficiency.
- [Loop Detection](../concepts/loop-detection.md) — Mechanisms for detecting and breaking infinite loops or unproductive cycles in agent execution, including lifecycle hooks for intervention.
- [Multi-hop RAG](../concepts/multihop-rag.md) — RAG approaches requiring multiple retrieval steps to answer complex questions, chaining retrievals across documents to build complete answers.
- [Dynamic Cheatsheet](../concepts/dynamic-cheatsheet.md) — An adaptive reference document that agents maintain and update with learned heuristics, patterns, and useful information during task execution.
- [Continual RAG](../concepts/continual-rag.md) — RAG systems that continuously update their knowledge bases with new information while maintaining retrieval performance without full reindexing.
- [Zero-Data Self-Evolution](../concepts/zero-data-self-evolution.md) — A self-improvement approach where agents evolve their capabilities without external training data, relying on self-generated experience and evaluation.
- [Tool-Integrated Reasoning](../concepts/tool-integrated-reasoning.md) — A reasoning paradigm where LLM agents interleave reasoning steps with tool calls, enabling grounded and verifiable problem solving.
- [Lossless Context Management](../concepts/lossless-context-management.md) — A context management approach that preserves all relevant information when handling long contexts, avoiding information loss through intelligent summarization and retrieval.
- [Agent Learning](../concepts/agent-learning.md) — The processes by which agents acquire new knowledge and capabilities through interaction, feedback, and experience, including in-context and parametric learning approaches.
- [Agentic RAG](../concepts/agentic-rag.md) — A RAG variant where agents dynamically plan and execute multi-step retrieval strategies, adapting queries based on intermediate results.
- [Cypher](../concepts/cypher.md) — A declarative graph query language used with Neo4j and other graph databases for querying knowledge graphs in agent memory systems.
- [Context Database](../concepts/context-database.md) — A specialized database for storing and retrieving agent context, enabling persistent and queryable context management across sessions.
- [Token Efficiency](../concepts/token-efficiency.md) — Strategies and metrics for minimizing token usage in agent interactions while maintaining task performance, including attention budget management.
- [Multi-Agent Collaboration](../concepts/multi-agent-collaboration.md) — Patterns and protocols for multiple AI agents working together, sharing information, and coordinating actions toward common goals.
- [KV Cache](../concepts/kv-cache.md) — Key-Value cache used in transformer inference to store intermediate attention computations, enabling faster generation by avoiding redundant computation for previously processed tokens.
- [Block Masking](../concepts/block-masking.md) — A technique in transformer attention that selectively masks entire blocks of tokens, used for efficient context management or training objectives like memory segmentation.
- [Memento (Concept)](../concepts/memento-concept.md) — A conceptual framework for agent memory inspired by the film Memento, emphasizing backward-chaining recall and the use of external memory artifacts to compensate for limited in-context memory.
- [Supervised Fine-Tuning](../concepts/supervised-fine-tuning.md) — A training approach where a pre-trained language model is further trained on labeled examples to improve performance on specific tasks or domains.
- [Dynamic Programming](../concepts/dynamic-programming.md) — An algorithmic optimization technique that solves complex problems by breaking them into overlapping subproblems and storing solutions, applied in agent planning and memory retrieval optimization.
- [AIME 2026](../concepts/aime26.md) — The 2026 American Invitational Mathematics Examination, used as a benchmark for evaluating mathematical reasoning capabilities of language models.
- [GPQA-Diamond](../concepts/gpqa-d.md) — The Diamond subset of the Graduate-Level Google-Proof Q&A benchmark, consisting of extremely challenging expert-level science questions used to evaluate frontier model reasoning.
- [LiveCodeBench](../concepts/lcb.md) — A benchmark for evaluating code generation capabilities of language models using live, continuously updated coding problems to prevent data contamination.
