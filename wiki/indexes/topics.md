# Topics Index

- [Retrieval-Augmented Generation](../concepts/rag.md) — A technique that enhances LLM outputs by retrieving relevant documents from an external knowledge base before generating a response. Combines retrieval systems with generative models to ground outputs in factual content.
- [Model Context Protocol](../concepts/mcp.md) — An open protocol developed by Anthropic that standardizes how AI models connect to external tools, data sources, and services. Enables LLMs to interact with the world through a common interface.
- [Episodic Memory](../concepts/episodic-memory.md) — A type of memory in cognitive science and AI systems that stores specific experiences or events in sequence. In agent systems, it refers to the ability to recall past interactions or task episodes.
- [Context Engineering](../concepts/context-engineering.md) — The discipline of designing, managing, and optimizing the information provided to LLMs within their context window. Goes beyond prompt engineering to encompass dynamic context construction, retrieval, and compression.
- [Agent Skills](../concepts/agent-skills.md) — Reusable, modular capabilities or procedures that agents can learn, store, and invoke to accomplish tasks. Central to self-improving agent architectures like Voyager and skill-based memory systems.
- [Semantic Memory](../concepts/semantic-memory.md) — A memory type that stores general factual knowledge and concepts independent of specific experiences. In agent systems, it enables recall of domain knowledge and world facts across sessions.
- [skill.md](../concepts/skill-md.md) — A markdown-based convention for storing and retrieving agent skills as structured text files. Used in coding agents and memory systems to persist reusable procedures.
- [ReAct](../concepts/react.md) — A prompting framework that interleaves reasoning and acting steps in LLM agents, allowing them to think step-by-step while taking actions and observing results.
- [BM25](../concepts/bm25.md) — A classical sparse retrieval ranking function widely used in information retrieval systems. Often combined with vector search in hybrid retrieval pipelines for RAG systems.
- [Reflexion](../concepts/reflexion.md) — A framework that enables LLM agents to improve through verbal self-reflection on past failures. Agents generate textual feedback and store it in memory to guide future attempts.
- [Progressive Disclosure](../concepts/progressive-disclosure.md) — A context management strategy that reveals information incrementally based on relevance and need. Applied in agent systems to manage context window limits by loading details on demand.
- [Knowledge Graph](../concepts/knowledge-graph.md) — A structured representation of entities and their relationships used for knowledge storage and retrieval. Increasingly used as a memory backend for AI agents to enable multi-hop reasoning.
- [Prompt Engineering](../concepts/prompt-engineering.md) — The practice of crafting and optimizing input prompts to elicit desired outputs from LLMs. Foundational to context engineering but superseded by more dynamic approaches in agentic systems.
- [Vector Database](../concepts/vector-database.md) — A database optimized for storing and retrieving high-dimensional vector embeddings. Core infrastructure for semantic search and RAG systems.
- [GRPO](../concepts/grpo.md) — Group Relative Policy Optimization, a reinforcement learning algorithm used to train LLMs with reward signals. Used in self-improving agent systems.
- [Procedural Memory](../concepts/procedural-memory.md) — Memory that stores how to perform tasks or procedures, analogous to muscle memory in humans. In agent systems, it encodes reusable skills and action sequences.
- [Hybrid Retrieval](../concepts/hybrid-retrieval.md) — A retrieval strategy combining sparse (BM25) and dense (vector) search methods to improve recall and precision. Commonly used in production RAG systems.
- [Semantic Search](../concepts/semantic-search.md) — A search approach that uses embedding models to find semantically similar content rather than exact keyword matches. Foundational to modern RAG and memory retrieval systems.
- [Temporal Reasoning](../concepts/temporal-reasoning.md) — The ability of AI systems to reason about time, sequences of events, and temporal relationships between entities. Critical for long-term memory and knowledge graph systems.
- [Zettelkasten](../concepts/zettelkasten.md) — A personal knowledge management methodology using atomic, interlinked notes developed by sociologist Niklas Luhmann. Influential in designing AI knowledge base structures.
- [agents.md](../concepts/agents-md.md) — A markdown convention for defining agent behavior, capabilities, and instructions. Used in coding agent frameworks to configure agent personas and guidelines.
- [Leiden Algorithm](../concepts/leiden-algorithm.md) — A community detection algorithm used in graph-based knowledge systems including GraphRAG to identify clusters of related entities in knowledge graphs.
- [Chain-of-Thought](../concepts/chain-of-thought.md) — A prompting technique that encourages LLMs to generate intermediate reasoning steps before producing a final answer, improving performance on complex tasks.
- [Core Memory](../concepts/core-memory.md) — A persistent in-context memory store in agent architectures like MemGPT/Letta that keeps essential information always available in the context window.
- [Context Window Management](../concepts/context-window-management.md) — Techniques and strategies for efficiently utilizing the limited context window of LLMs, including compression, summarization, and selective loading of information.
- [Multi-Hop Reasoning](../concepts/multi-hop-reasoning.md) — Reasoning that requires connecting information across multiple documents or knowledge graph edges to answer complex questions. A key challenge for RAG and knowledge graph systems.
- [Agentic RAG](../concepts/agentic-rag.md) — An advanced RAG paradigm where an agent dynamically decides when and how to retrieve information, potentially iterating multiple retrieval-generation cycles.
- [Reinforcement Learning](../concepts/reinforcement-learning.md) — A machine learning paradigm where agents learn by interacting with environments and receiving reward signals. Increasingly applied to LLM fine-tuning and self-improvement.
- [Entity Extraction](../concepts/entity-extraction.md) — The process of automatically identifying and classifying named entities from text. A core step in building knowledge graphs and structured memory systems from unstructured data.
- [In-Context Learning](../concepts/in-context-learning.md) — The ability of LLMs to learn from examples provided within the prompt context without parameter updates. A key mechanism for few-shot adaptation.
- [Continual Learning](../concepts/continual-learning.md) — The ability of AI systems to continuously learn new information without forgetting previously learned knowledge. Related to catastrophic forgetting and lifelong learning.
- [Eval-Driven Development](../concepts/eval-driven-development.md) — A development methodology that uses systematic evaluation and benchmarking to guide agent improvement cycles, analogous to test-driven development in software engineering.
- [Godel Machine](../concepts/godel-machine.md) — A theoretical self-referential general problem solver that rewrites itself when it can prove the rewrite will improve performance. Inspiration for modern self-improving agent architectures.
- Prompt Injection — A security attack where malicious content in retrieved data or tool outputs manipulates agent behavior by overriding instructions. A key security concern in RAG and agentic systems.
- [Synthetic Data Generation](../concepts/synthetic-data-generation.md) — The process of automatically creating training or evaluation data using LLMs or other generative methods. Used to bootstrap skill libraries and improve agent capabilities.
- Bi-Temporal Indexing — A data modeling approach that tracks both the time when facts were valid in the real world and when they were recorded in the system, enabling accurate historical queries.
- [Case-Based Reasoning](../concepts/case-based-reasoning.md) — A problem-solving approach that reuses solutions from similar past cases. Applied in agent memory systems to retrieve and adapt previous solutions to new problems.
- [Long-Term Memory](../concepts/long-term-memory.md) — Persistent memory that spans multiple sessions or interactions in AI agent systems. Encompasses episodic, semantic, and procedural memory stores.
- [Chunking](../concepts/chunking.md) — The process of splitting documents into smaller segments for indexing and retrieval in RAG systems. Chunking strategy significantly affects retrieval quality.
- [Multi-Agent Systems](../concepts/multi-agent-systems.md) — Architectures where multiple AI agents collaborate, compete, or specialize to accomplish complex tasks that exceed individual agent capabilities.
- Knowledge Distillation — A model compression technique where a smaller student model is trained to mimic a larger teacher model. Applied in self-improving systems to transfer capabilities.
- [Automatic Curriculum](../concepts/dspy-optimizer.md) — A self-improving mechanism that automatically generates and adjusts training tasks based on agent performance, enabling progressive skill acquisition without manual curriculum design.
