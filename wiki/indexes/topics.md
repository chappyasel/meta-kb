# Topics Index

- [Retrieval-Augmented Generation](../concepts/rag.md) — A technique that augments LLM generation by retrieving relevant documents or passages from an external knowledge store at inference time. Foundational approach in LLM knowledge bases.
- [Model Context Protocol](../concepts/mcp.md) — An open protocol for connecting LLMs to external tools, data sources, and services. Enables agents to access and manipulate context programmatically.
- [Episodic Memory](../concepts/episodic-memory.md) — A form of agent memory that stores records of specific past experiences or interactions, analogous to human episodic memory. Enables agents to recall and learn from prior events.
- [GraphRAG](../concepts/graphrag.md) — A retrieval-augmented generation approach that uses knowledge graphs to structure and retrieve information, enabling more complex multi-hop reasoning than flat vector retrieval.
- [Semantic Memory](../concepts/semantic-memory.md) — A type of agent memory that stores general world knowledge and facts, distinct from episodic or procedural memory. Enables agents to recall stable factual knowledge across sessions.
- [ReAct](../concepts/react.md) — A prompting and agent framework that interleaves reasoning and acting steps, allowing agents to think through problems while using tools iteratively.
- [Context Engineering](../concepts/context-engineering.md) — The discipline of designing, managing, and optimizing the information provided to LLMs in their context window to improve task performance. Encompasses prompt engineering, context compression, and retrieval strategies.
- [BM25](../concepts/bm25.md) — A classical term-frequency-based ranking function used in information retrieval and hybrid search systems alongside vector embeddings.
- [Progressive Disclosure](../concepts/progressive-disclosure.md) — A context management strategy where information is revealed to an agent incrementally based on need, reducing context bloat and improving relevance.
- [Vector Database](../concepts/vector-database.md) — A database optimized for storing and querying high-dimensional vector embeddings, enabling semantic similarity search for RAG and agent memory systems.
- [Agent Skills](../concepts/agent-skills.md) — Reusable, learnable capabilities that agents can acquire, store, and retrieve to accomplish tasks more efficiently over time. Stored as skill files or in a skill library.
- [skill.md](../concepts/skill-md.md) — A markdown-based convention for storing agent skills and procedural knowledge as structured files that agents can read and execute.
- [Reflexion](../concepts/reflexion.md) — A framework for self-improving agents that uses verbal reinforcement and reflective feedback to improve performance over trials without gradient updates.
- [Prompt Engineering](../concepts/prompt-engineering.md) — The practice of crafting and optimizing natural language prompts to elicit desired behaviors from LLMs, a precursor to and component of context engineering.
- [Agent Memory](../concepts/agent-memory.md) — The collective systems and mechanisms that enable AI agents to store, retrieve, and utilize information across interactions, including episodic, semantic, procedural, and core memory types.
- [claude.md](../concepts/claude-md.md) — A markdown file convention used to provide persistent instructions, context, and memory to Claude agents across sessions, analogous to a system prompt stored on disk.
- [Self-Improving Agents](../concepts/self-improving-agents.md) — AI agents that can modify their own behavior, skills, or code over time based on experience, feedback, or reflection, enabling continuous improvement without human intervention.
- [GRPO](../concepts/grpo.md) — Group Relative Policy Optimization, a reinforcement learning algorithm used to train and fine-tune language models, particularly for reasoning and agent tasks.
- [Knowledge Graph](../concepts/knowledge-graph.md) — A graph-based representation of entities and their relationships used to structure knowledge for retrieval, reasoning, and agent memory systems.
- [Procedural Memory](../concepts/procedural-memory.md) — A type of agent memory that stores learned procedures, workflows, and skills that agents can execute to accomplish tasks.
- [GEPA](../concepts/gepa.md) — A self-improving agent framework based on genetic or evolutionary algorithms for prompt and agent optimization.
- [Core Memory](../concepts/core-memory.md) — Always-present, immediately accessible memory in an agent's context window, used for storing critical facts about the user and agent persona.
- [Context Management](../concepts/context-management.md) — The set of strategies and systems for managing what information is included in an LLM's context window, including compression, retrieval, and windowing techniques.
- [Context Compression](../concepts/context-compression.md) — Techniques for reducing the size of LLM context while preserving relevant information, including prompt compression and summarization methods.
- [Hybrid Retrieval](../concepts/hybrid-retrieval.md) — A retrieval strategy combining dense vector search with sparse keyword-based search (e.g., BM25) to improve recall and precision in RAG systems.
- [Darwin Gödel Machine](../concepts/darwin-godel-machine.md) — A self-improving AI system concept that uses evolutionary and formal verification approaches to modify its own code, drawing on Gödel machine theory.
- [Cognitive Architecture](../concepts/cognitive-architecture.md) — The overall design of an agent's memory, reasoning, and action systems, often inspired by cognitive science models of human cognition.
- [Continual Learning](../concepts/continual-learning.md) — Machine learning approaches that enable models or agents to learn from new data over time without forgetting previously acquired knowledge.
- [Reinforcement Learning](../concepts/reinforcement-learning.md) — A machine learning paradigm where agents learn to take actions by receiving rewards or penalties, used in training and self-improving agent systems.
- [Chain-of-Thought](../concepts/chain-of-thought.md) — A prompting technique that elicits step-by-step reasoning from LLMs by including reasoning chains in the prompt or context.
- [Multi-Agent Systems](../concepts/multi-agent-systems.md) — Systems composed of multiple AI agents that coordinate, collaborate, or compete to accomplish complex tasks, requiring shared or distributed memory and context.
- [Community Detection](../concepts/community-detection.md) — Graph algorithms used to identify clusters or communities in knowledge graphs, supporting hierarchical summarization and retrieval in GraphRAG systems.
- [Personalized PageRank](../concepts/personalized-pagerank.md) — A graph traversal algorithm used in knowledge graph retrieval to rank nodes by relevance to a query, used in HippoRAG and similar systems.
- [Zettelkasten](../concepts/zettelkasten.md) — A note-taking and knowledge management methodology using atomic, linked notes, influential in personal knowledge management and LLM knowledge base design.
- [Case-Based Reasoning](../concepts/case-based-reasoning.md) — A problem-solving approach where agents recall and adapt solutions from past similar cases, applicable to episodic memory systems.
- [Multi-Hop Reasoning](../concepts/multi-hop-reasoning.md) — The ability to answer questions or solve problems by chaining multiple retrieval or inference steps across a knowledge base or graph.
- [Execution Traces](../concepts/execution-traces.md) — Records of an agent's action sequences and decision paths during task execution, used for memory, skill extraction, and self-improvement.
- [Synthetic Data Generation](../concepts/synthetic-data-generation.md) — The process of generating artificial training or evaluation data using LLMs or other methods to augment agent learning and skill development.
- [Git as Memory](../concepts/git-as-memory.md) — Using version control systems like git as a structured memory layer for agents, tracking changes, decisions, and code evolution over time.
- [Dynamic Cheatsheet](../concepts/dynamic-cheatsheet.md) — A context engineering pattern where an agent maintains an evolving reference document of key facts, rules, and learned heuristics in its context.
- [LLM-as-a-Judge](../concepts/llm-as-judge.md) — Using a language model to evaluate the quality of outputs from another model or agent, enabling automated quality assessment and reward signaling.
- [Meta-Harness](../concepts/meta-harness.md) — A system for automatically optimizing the scaffolding, prompts, and tools used by an LLM agent, treating the harness itself as an optimizable component.
- [Context Graph](../concepts/context-graph.md) — A graph-based representation of context relationships and dependencies used to manage and navigate complex agent contexts.
- [Organizational Memory](../concepts/organizational-memory.md) — The collective knowledge, processes, and context stored and shared across an organization or multi-agent system, enabling institutional learning.
- [Bounded Context](../concepts/bounded-context.md) — A software architecture pattern applied to agent systems to define clear boundaries around context domains, preventing context pollution and improving modularity.
- [Compaction Tree](../concepts/compaction-tree.md) — A hierarchical summarization structure used to compress long conversation histories or context into progressively more abstract representations.
- [Automatic Curriculum](../concepts/automatic-curriculum.md) — A method for automatically generating increasingly difficult training tasks or experiences for self-improving agents, enabling progressive skill acquisition.
- [Credit Assignment](../concepts/credit-assignment.md) — The problem of determining which actions or decisions in an agent's trajectory were responsible for a final outcome, critical for self-improvement and reinforcement learning.
- [Compositional Skill Synthesis](../concepts/compositional-skill-synthesis.md) — An approach to building agent capabilities by combining simpler learned skills into more complex composite skills.
- [Ebbinghaus Forgetting Curve](../concepts/ebbinghaus-forgetting-curve.md) — A psychological model of memory decay over time, used as inspiration for spaced repetition and memory management in agent systems.
- [Temporal Reasoning](../concepts/temporal-reasoning.md) — The ability of agents to reason about time, temporal relationships between events, and the recency or validity of stored knowledge.
- [Markdown Wiki](../concepts/markdown-wiki.md) — A knowledge base built from interconnected markdown files, used as a lightweight LLM-maintainable knowledge store following the Karpathy wiki pattern.
- [Reward Hacking](../concepts/reward-hacking.md) — A failure mode in reinforcement learning where agents exploit loopholes in reward functions rather than learning intended behaviors.
- [Supervised Fine-Tuning](../concepts/sft.md) — A training technique for adapting pre-trained language models to specific tasks using labeled examples, often combined with RLHF or GRPO.
