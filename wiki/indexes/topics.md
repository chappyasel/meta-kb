# Topics Index

- [Retrieval-Augmented Generation](../concepts/retrieval-augmented-generation.md) — A technique combining information retrieval with language model generation, where relevant documents are fetched from a knowledge base and injected into the context before generating a response. Includes agentic RAG variants.
- [Episodic Memory](../concepts/episodic-memory.md) — A type of agent memory that stores specific past experiences or events, enabling agents to recall and reason about prior interactions and outcomes.
- [Context Engineering](../concepts/context-engineering.md) — The discipline of designing, managing, and optimizing the information provided in an LLM's context window to improve agent performance. Encompasses prompt engineering, context compression, and memory retrieval strategies.
- [Model Context Protocol](../concepts/model-context-protocol.md) — A protocol developed by Anthropic standardizing how context and tool results are passed to language models in agent systems. Enables interoperability between agent components and external tools.
- [Knowledge Graph](../concepts/knowledge-graph.md) — A structured representation of entities and their relationships used as a knowledge substrate for agent reasoning and retrieval. Used in systems like GraphRAG and Graphiti.
- [ReAct](../concepts/react.md) — A prompting and agent architecture paradigm combining reasoning and acting, where agents interleave chain-of-thought reasoning with tool use actions.
- [Semantic Memory](../concepts/semantic-memory.md) — A type of agent memory storing general factual knowledge and concepts, distinct from episodic (event-based) or procedural (skill-based) memory.
- [Agent Memory](../concepts/agent-memory.md) — The general concept of how AI agents store, retrieve, and manage information across interactions, encompassing episodic, semantic, procedural, and core memory types.
- [Agent Skills](../concepts/agent-skills.md) — Reusable capabilities or procedures that agents can invoke to accomplish tasks, often stored in skill libraries and composed to handle complex workflows.
- [Progressive Disclosure](../concepts/progressive-disclosure.md) — A context engineering strategy that reveals information incrementally based on relevance and need, avoiding context overload by surfacing details only when required.
- [Long-Term Memory](../concepts/long-term-memory.md) — Persistent memory that agents maintain across sessions and interactions, enabling continuity and accumulation of knowledge over time.
- [Reflexion](../concepts/reflexion.md) — An agent learning framework where agents reflect on their past actions and failures using verbal reinforcement, storing reflections in memory to improve future performance.
- [GEPA](../concepts/gepa.md) — A self-improving agent framework focused on automated program improvement and evolutionary agent optimization approaches.
- [CLAUDE.md](../concepts/claude-md.md) — A special markdown file used to provide persistent instructions, context, and agent configuration to Claude-based coding agents across sessions.
- [GRPO](../concepts/grpo.md) — Group Relative Policy Optimization, a reinforcement learning algorithm used for fine-tuning language models, particularly in self-improving agent contexts.
- [Semantic Search](../concepts/semantic-search.md) — Search that uses embedding-based similarity to find semantically relevant documents rather than relying on exact keyword matching.
- [Context Management](../concepts/context-management.md) — Strategies and systems for managing what information occupies an agent's context window, including compression, eviction, and retrieval decisions.
- [Procedural Memory](../concepts/procedural-memory.md) — Agent memory encoding skills, workflows, and action sequences — how to perform tasks rather than what facts are true.
- [Multi-Agent Systems](../concepts/multi-agent-systems.md) — Architectures involving multiple coordinating AI agents that collaborate, delegate, and communicate to accomplish complex tasks.
- [Vector Database](../concepts/vector-database.md) — A database optimized for storing and querying high-dimensional vector embeddings, used as the retrieval backbone in RAG and agent memory systems.
- [BM25](../concepts/bm25.md) — A classical sparse retrieval algorithm based on term frequency and inverse document frequency, commonly combined with dense retrieval in hybrid search systems.
- [Human-in-the-Loop](../concepts/human-in-the-loop.md) — An agent design pattern where human oversight, feedback, or approval is incorporated into the agent's decision-making or execution pipeline.
- [Continual Learning](../concepts/continual-learning.md) — The ability of agents to learn and adapt from new experiences without forgetting previously acquired knowledge, addressing catastrophic forgetting challenges.
- [Execution Traces](../concepts/execution-traces.md) — Records of an agent's step-by-step actions, decisions, and observations during task execution, used for debugging, evaluation, and learning.
- [Lost in the Middle](../concepts/lost-in-the-middle.md) — The phenomenon where LLMs struggle to recall information placed in the middle of long contexts, motivating research into context organization and retrieval strategies.
- [Self-Improving Agents](../concepts/self-improving-agents.md) — AI agents capable of autonomously improving their own capabilities, prompts, code, or architectures through experience, reflection, and optimization loops.
- [Core Memory](../concepts/core-memory.md) — A fixed-size, always-present memory block in agent architectures (as in MemGPT/Letta) containing essential persistent information about the user and agent persona.
- [Community Detection](../concepts/community-detection.md) — Graph algorithms (such as Leiden) used to identify clusters of related entities in knowledge graphs, enabling hierarchical summarization and navigation in systems like GraphRAG.
- [Context Graphs](../concepts/context-graphs.md) — Graph-based representations of contextual information and relationships between concepts, used to structure and navigate agent context.
- [Context Compression](../concepts/context-compression.md) — Techniques for reducing the token length of context while preserving relevant information, including prompt compression methods like LLMLingua.
- [Cognitive Architecture](../concepts/cognitive-architecture.md) — The overall structural design of an AI agent's mental processes, including perception, memory, reasoning, and action components inspired by cognitive science.
- [Hybrid Search](../concepts/hybrid-search.md) — A retrieval strategy combining sparse (BM25) and dense (vector) search methods to improve recall and precision in knowledge retrieval systems.
- [Temporal Reasoning](../concepts/temporal-reasoning.md) — The ability to reason about time-ordered events, timestamps, and temporal relationships in agent memory and knowledge retrieval.
- [Reinforcement Learning](../concepts/reinforcement-learning.md) — A learning paradigm where agents improve by receiving reward signals from their environment, used in agent fine-tuning and self-improvement loops.
- [SkillBook](../concepts/skill-book.md) — A structured repository or library of agent skills and capabilities, enabling agents to look up and invoke learned procedures.
- [Chain-of-Thought](../concepts/chain-of-thought.md) — A prompting technique where models generate intermediate reasoning steps before producing a final answer, improving performance on complex tasks.
- [Synthetic Data Generation](../concepts/synthetic-data-generation.md) — The process of generating training or evaluation data using AI models, used in agent self-improvement and skill acquisition pipelines.
- [Zettelkasten](../concepts/zettelkasten.md) — A knowledge management method using interconnected atomic notes with unique identifiers and bidirectional links, influencing agent knowledge organization approaches.
- [Prompt Optimization](../concepts/prompt-optimization.md) — Automated methods for improving LLM prompts through search, gradient-based methods, or LLM feedback to enhance agent performance.
- [Organizational Memory](../concepts/organizational-memory.md) — Collective knowledge and procedures stored and accessible across an organization or multi-agent system, enabling institutional learning and continuity.
- [Case-Based Reasoning](../concepts/case-based-reasoning.md) — A problem-solving paradigm where agents reuse solutions from similar past cases stored in memory to address new problems.
- [Workflow Induction](../concepts/workflow-induction.md) — The process of automatically inferring reusable workflow templates or procedures from agent execution traces or demonstrations.
- [Incremental Indexing](../concepts/incremental-indexing.md) — The practice of continuously updating knowledge indices as new information arrives, rather than batch reindexing, enabling real-time knowledge base updates.
- [Short-Term Memory](../concepts/short-term-memory.md) — Temporary in-context storage used by agents during a single session or task, corresponding to the active context window contents.
- [LLM-as-Judge](../concepts/llm-as-judge.md) — Using a language model to evaluate the quality of agent outputs, enabling automated assessment without human annotators in optimization loops.
- [Personalized PageRank](../concepts/personalized-pagerank.md) — A graph traversal algorithm adapted for knowledge retrieval that propagates relevance scores from query-matched entities through a knowledge graph.
- [Entity Extraction](../concepts/entity-extraction.md) — The process of identifying and extracting named entities and their relationships from text to populate knowledge graphs and memory systems.
- [Agent Harness](../concepts/agent-harness.md) — The scaffolding infrastructure surrounding an agent including evaluation loops, tool integrations, sandbox environments, and optimization components.
- [Docker Sandbox](../concepts/docker-sandbox.md) — Containerized execution environments used to safely run agent-generated code and commands in isolation from the host system.
- [Observability](../concepts/observability.md) — Systems and practices for monitoring, tracing, and understanding agent behavior in production, including logging, metrics, and trace collection.
- [Memory Evolution](../concepts/memory-evolution.md) — The process by which agent memory structures are dynamically updated, refined, and reorganized over time as new information and experiences accumulate.
- [World Model](../concepts/world-model.md) — An internal representation an agent maintains of its environment, enabling planning and prediction of action consequences.
- [Compositional Skill Synthesis](../concepts/composable-skills.md) — The automated generation and composition of agent skills from simpler primitives or demonstrations to handle novel complex tasks.
- [Markdown Wiki](../concepts/markdown-wiki.md) — A knowledge base built from interlinked markdown files, used as a lightweight knowledge substrate for agent systems and personal knowledge management.
- [System Prompt](../concepts/system-prompt.md) — Instructions provided to an LLM at the beginning of a context to define its behavior, persona, and capabilities for a given agent deployment.
- [Meta-Agent](../concepts/meta-agent.md) — An agent that orchestrates, evaluates, or improves other agents, operating at a higher level of abstraction in multi-agent hierarchies.
- [Tool Registry](../concepts/tool-registry.md) — A catalog of available tools and their specifications that agents can query to discover and invoke capabilities dynamically.
- [Structured Output](../concepts/structured-output.md) — LLM output formatted according to a defined schema (e.g., JSON, Pydantic models), enabling reliable parsing and integration in agent pipelines.
- [Context Generation](../concepts/abductive-context.md) — The process of dynamically constructing or generating relevant context for an agent's current task from available knowledge sources.
- [Agent-Mediated Workflows](../concepts/agent-mediated-workflows.md) — Business and software processes where AI agents take an active role in orchestrating, executing, and managing workflow steps.
- [Compound Engineering](../concepts/compound-engineering.md) — An approach to building AI systems that combines multiple specialized components (models, retrievers, tools) into integrated compound AI systems.
- [Bounded Context](../concepts/bounded-context.md) — A design principle from domain-driven design applied to agent systems, defining clear boundaries for what context and knowledge is relevant to a given agent or task.
- [Loop Detection](../concepts/loop-detection.md) — Mechanisms for identifying when an agent is stuck in repetitive or circular behavior, enabling graceful recovery or escalation.
- [Multi-hop RAG](../concepts/multihop-rag.md) — Retrieval-augmented generation requiring multiple sequential retrieval steps to gather information spread across documents for complex question answering.
- [Dynamic Cheatsheet](../concepts/dynamic-cheatsheet.md) — A dynamically updated reference document maintained by an agent containing learned heuristics, facts, and strategies for improving task performance.
- [Continual RAG](../concepts/continual-rag.md) — RAG systems that continuously update their knowledge indices with new information, combining retrieval with ongoing learning.
- [Zero-Data Self-Evolution](../concepts/zero-data-self-evolution.md) — An approach to agent improvement that requires no external training data, relying on the agent's own experience and self-generated feedback.
- [Tool-Integrated Reasoning](../concepts/tool-integrated-reasoning.md) — A reasoning paradigm where agents interleave natural language reasoning with tool calls, enabling grounded and verifiable problem-solving.
