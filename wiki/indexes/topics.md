# Topics Index

- [Retrieval-Augmented Generation](../concepts/rag.md) — A technique that enhances LLM responses by retrieving relevant documents or passages from an external knowledge base before generating an answer. Combines the strengths of retrieval systems with generative models.
- [Episodic Memory](../concepts/episodic-memory.md) — A type of agent memory that stores specific past experiences or interaction episodes, enabling agents to recall and leverage prior events. Analogous to human episodic memory.
- [Model Context Protocol](../concepts/mcp.md) — Anthropic's open protocol for connecting AI models to external tools, data sources, and services. Standardizes how LLM agents communicate with context providers and tool servers.
- [Agent Skills](../concepts/agent-skills.md) — Reusable, composable capabilities or procedures that agents can learn, store, and invoke to accomplish tasks. Often stored as skill libraries or registries for retrieval.
- [Semantic Memory](../concepts/semantic-memory.md) — A type of agent memory storing general facts, concepts, and world knowledge rather than specific episodes. Analogous to human long-term declarative knowledge.
- [GraphRAG](../concepts/graphrag.md) — A retrieval-augmented generation approach that uses knowledge graphs to structure and retrieve information, enabling multi-hop reasoning over interconnected entities. Originally developed by Microsoft.
- [ReAct](../concepts/react.md) — A prompting framework for LLM agents that interleaves reasoning and acting steps, enabling agents to think through problems while taking actions and observing results.
- [Context Engineering](../concepts/context-engineering.md) — The discipline of designing and managing the information provided to LLMs in their context window, including what to include, how to structure it, and how to compress or retrieve it dynamically.
- [BM25](../concepts/bm25.md) — A classic keyword-based ranking algorithm widely used in information retrieval and search systems. Often combined with vector search for hybrid retrieval in RAG systems.
- [Knowledge Graph](../concepts/knowledge-graph.md) — A structured representation of entities and their relationships, used to store and query factual knowledge. Increasingly used as a memory and retrieval layer for LLM agents.
- [Progressive Disclosure](../concepts/progressive-disclosure.md) — A context management technique that reveals information to an agent incrementally based on need, rather than loading all context upfront, to manage context window constraints.
- [Vector Database](../concepts/vector-database.md) — A database system optimized for storing and querying high-dimensional vector embeddings, enabling semantic similarity search. Core infrastructure for RAG and agent memory systems.
- [Reflexion](../concepts/reflexion.md) — A framework where LLM agents reflect on their past trajectories and failures to improve future performance through verbal reinforcement learning, without updating model weights.
- [GEPA](../concepts/gepa.md) — An approach for generative evolutionary prompt/agent optimization that uses LLMs to evolve and improve agent capabilities or prompts over time.
- [Continual Learning](../concepts/continual-learning.md) — Machine learning paradigm where models learn from a continuous stream of data without forgetting previously learned knowledge. Key challenge is catastrophic forgetting.
- [GRPO](../concepts/grpo.md) — Group Relative Policy Optimization, a reinforcement learning algorithm for fine-tuning LLMs that uses group-based reward estimation instead of a separate critic model.
- [Agentic RAG](../concepts/agentic-rag.md) — An extension of RAG where agents actively plan retrieval strategies, iterate on search queries, and synthesize information from multiple sources using agentic reasoning.
- [Procedural Memory](../concepts/procedural-memory.md) — A type of agent memory storing how-to knowledge and learned procedures for accomplishing tasks. Analogous to human skill memory, often represented as reusable skill programs.
- [Prompt Engineering](../concepts/prompt-engineering.md) — The practice of crafting and optimizing prompts to elicit desired behaviors from LLMs. A foundational technique for context engineering and agent system design.
- [Agent Memory](../concepts/agent-memory.md) — The collective mechanisms by which AI agents store, retrieve, and utilize information across interactions, including episodic, semantic, procedural, and core memory types.
- [Core Memory](../concepts/core-memory.md) — A memory type in MemGPT/Letta representing the agent's always-present context including persona, user information, and key facts that persist across all interactions.
- [Self-Improving Agent](../concepts/self-improving-agent.md) — An AI agent capable of modifying its own capabilities, strategies, or knowledge based on experience and feedback, without direct human intervention.
- [Karpathy Loop](../concepts/karpathy-loop.md) — An iterative improvement cycle attributed to Andrej Karpathy where agents or systems continually evaluate, identify weaknesses, and improve through repeated cycles of testing and refinement.
- [CLAUDE.md](../concepts/claude-md.md) — A special configuration file used with Claude Code that provides persistent instructions, project context, and behavioral guidelines to the agent across sessions.
- [Hybrid Search](../concepts/hybrid-search.md) — A retrieval strategy that combines dense vector search with sparse keyword search (BM25) to leverage the strengths of both approaches for more robust document retrieval.
- [Context Compression](../concepts/context-compression.md) — Techniques for reducing the size of context provided to LLMs while preserving essential information, including summarization, distillation, and prompt compression methods.
- [Context Window](../concepts/context-window.md) — The maximum amount of text an LLM can process in a single inference call. Managing and optimizing context window usage is central to context engineering.
- [Reinforcement Learning](../concepts/reinforcement-learning.md) — A machine learning paradigm where agents learn to maximize rewards through trial and error interaction with an environment, increasingly applied to improve LLM agent capabilities.
- [Chain-of-Thought](../concepts/chain-of-thought.md) — A prompting technique that encourages LLMs to produce step-by-step reasoning before giving a final answer, improving performance on complex reasoning tasks.
- [Task Decomposition](../concepts/task-decomposition.md) — The process of breaking complex tasks into smaller, manageable subtasks that can be assigned to agents or solved sequentially, a core pattern in multi-agent systems.
- [Embedding Model](../concepts/embedding-model.md) — A neural network that converts text or other data into dense vector representations for semantic similarity search and retrieval in RAG systems.
- [Meta-Agent](../concepts/meta-agent.md) — An agent that operates over other agents, managing, coordinating, or improving them. Used in automated agent design and optimization systems.
- [Skill Files](../concepts/skill-md.md) — Markdown-based files used to store reusable agent skills, procedures, and workflows for retrieval and execution during task completion.
- [LLM-as-Judge](../concepts/llm-as-judge.md) — A technique using LLMs to evaluate the quality of other LLM outputs, enabling automated evaluation pipelines without human annotation.
- [Catastrophic Forgetting](../concepts/catastrophic-forgetting.md) — The tendency of neural networks to lose previously learned knowledge when trained on new data, a fundamental challenge in continual learning for AI systems.
- [Synthetic Data Generation](../concepts/synthetic-data-generation.md) — The process of creating artificial training data using LLMs or other methods to improve model capabilities, often used in self-improvement pipelines.
- [Decision Traces](../concepts/decision-traces.md) — Records of agent reasoning steps, actions, and decisions during task execution, used for debugging, evaluation, and learning from past experience.
- [Workflow Induction](../concepts/workflow-induction.md) — The process of automatically discovering and codifying reusable workflow patterns from agent execution histories for future reuse.
- [Zettelkasten](../concepts/zettelkasten.md) — A personal knowledge management method using atomic, interlinked notes for building a networked knowledge base. Influential in personal PKM tools like Obsidian.
- [Memory Evolution](../concepts/memory-evolution.md) — The process of updating, consolidating, and improving agent memory over time through experience, including mechanisms for memory summarization, merging, and decay.
- [Case-Based Reasoning](../concepts/case-based-reasoning.md) — A problem-solving approach that reuses solutions from similar past cases, increasingly applied to agent memory systems for experience-based learning.
- [HNSW](../concepts/hnsw.md) — Hierarchical Navigable Small World graphs, an efficient algorithm for approximate nearest neighbor search used in vector databases for fast similarity retrieval.
- [Reciprocal Rank Fusion](../concepts/reciprocal-rank-fusion.md) — A rank aggregation method that combines results from multiple retrieval systems into a single ranked list, widely used in hybrid search pipelines.
- [Entity Extraction](../concepts/entity-extraction.md) — The process of identifying and extracting named entities and their relationships from text, foundational for building knowledge graphs and structured memory.
- [Context Management](../concepts/context-management.md) — Strategies and techniques for managing what information is included in an LLM's context window across multi-turn interactions, balancing relevance and length.
- [Human-in-the-Loop](../concepts/human-in-the-loop.md) — A design pattern where humans provide oversight, feedback, or approval at critical points in an AI agent's workflow to ensure correctness and safety.
- [Automatic Curriculum](../concepts/automatic-curriculum.md) — A mechanism for automatically generating progressively challenging tasks for agents to learn from, enabling open-ended skill acquisition without manual curriculum design.
- [Personalized PageRank](../concepts/personalized-pagerank.md) — A graph ranking algorithm variant used in knowledge graph retrieval to find relevant nodes based on proximity to query entities, used in HippoRAG.
- [Muon Optimizer](../concepts/muon-optimizer.md) — A novel optimizer for training neural networks using Nesterov momentum applied in the spectral domain, offering improved training dynamics.
- [Prompt Optimization](../concepts/dspy-optimization.md) — Automated techniques for optimizing prompts and context structures for LLMs, including DSPy compilation and other gradient-free optimization methods.
- [LLM Knowledge Base](../concepts/knowledge-base.md) — A structured repository of information designed to be queried and utilized by LLM agents, encompassing documents, facts, and relationships organized for AI consumption.
- [Emotional Memory](../concepts/emotional-memory.md) — A memory component that tracks affective states, emotional context, and sentiment associated with past interactions to inform agent behavior.
- [Unknown Unknowns](../concepts/unknown-unknowns.md) — Information gaps an agent is unaware of, representing a key challenge in knowledge base design and context engineering where missing knowledge isn't surfaced.
