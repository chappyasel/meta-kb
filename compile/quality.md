# Quality Criteria

## Citation rules

- Every factual claim MUST cite a specific source: `[Source](../raw/type/file.md)`
- Every project mentioned MUST link to its reference card: `[Mem0](projects/mem0.md)`
- If a claim can't be sourced from raw/, mark it as `[unverified]`

## Comparison rules

- Comparisons MUST include a recommendation: "Use X when Y, use Z when W"
- Don't say "both have pros and cons." Take a position.
- Back recommendations with evidence (star counts, benchmarks, adoption signals)

## Insight rules

- Surface non-obvious insights. If it's in every README, it's not an insight.
- Look for: patterns across sources, disagreements between approaches,
  things the community is converging on that aren't yet conventional wisdom
- "Everyone is moving toward hybrid episodic+semantic memory" is an insight.
  "Memory is important for agents" is not.

## Neutrality

- Equal treatment of all projects, including CAIK-adjacent ones
- Same depth AND same criticism for every project
- If a project has serious limitations, say so directly

## Tone

- Write for practitioners who build with AI agents daily
- Be direct and opinionated. Not academic, not corporate.
- Use concrete examples: project names, star counts, specific techniques
- Short paragraphs. Direct sentences.
