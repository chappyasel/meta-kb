# Project Index

> The project cards are grouped here by primary bucket. Many projects span multiple buckets, but this index places each one where its core contribution is clearest.

## Knowledge Bases

- [Ars Contexta](../projects/ars-contexta.md) builds domain-shaped note vaults instead of generic knowledge dumps.
- [Napkin](../projects/napkin.md) treats markdown files plus BM25 and staged reveal as a serious memory stack.
- [PageIndex](../projects/pageindex.md) indexes hierarchical document structure instead of flattening everything into chunks.
- [HippoRAG](../projects/hipporag.md) uses graph-style associative retrieval for synthesis-heavy questions.
- [Skill Seekers](../projects/skill-seekers.md) turns raw knowledge into portable outputs for many agent ecosystems.

## Agent Memory

- [Mem0](../projects/mem0.md) is the mainstream multi-level memory layer for user, session, and agent state.
- [Letta](../projects/letta.md) focuses on stateful agents with explicit memory blocks.
- [Memori](../projects/memori.md) stores extracted memory in a SQL-native structure with strong efficiency numbers.
- [Graphiti](../projects/graphiti.md) represents memory as a temporal knowledge graph with provenance.
- [Cognee](../projects/cognee.md) combines graph and vector retrieval into a continuously maintained memory layer.
- [Claude-Mem](../projects/claude-mem.md) uses file-based search and timeline memory for coding and operator workflows.
- [planning-with-files](../projects/planning-with-files.md) keeps durable working memory in markdown planning artifacts.
- [OpenViking](../projects/openviking.md) uses hierarchical filesystem memory with L0/L1/L2 layers.
- [Hipocampus](../projects/hipocampus.md) explores low-cost long-term recall across domains.
- [Nemori](../projects/nemori.md) argues for event-boundary memory instead of arbitrary chunking.
- [Mirix](../projects/mirix.md) formalizes multi-type memory across several distinct layers.
- [Memento](../projects/memento.md) retrieves prior cases to guide new decisions.
- [MemoryBank](../projects/memorybank.md) emphasizes forgetting, contradiction handling, and durable relationship memory.
- [Supermemory](../projects/supermemory.md) focuses on broad personal memory across interactions.

## Context Engineering

- [LLMLingua](../projects/llmlingua.md) is the compression-first answer to prompt bloat.
- [ACE](../projects/ace.md) evolves context playbooks instead of relying on a fixed prompt.
- [everything-claude-code](../projects/everything-claude-code.md) exposes context design as a large operational surface.

## Agent Systems

- [Anthropic Skills](../projects/anthropic-skills.md) define the reference `SKILL.md` capability format.
- [claude-skills](../projects/claude-skills.md) is a broad community registry of reusable skills.
- [Obsidian Skills](../projects/obsidian-skills.md) adapts skills to a notes-native environment.
- [AI-Research-SKILLs](../projects/ai-research-skills.md) package research workflows into reusable modules.
- [gstack](../projects/gstack.md) layers workflows, reviews, and safety modes on top of coding agents.

## Self-Improving Systems

- [Autoresearch](../projects/autoresearch.md) is the benchmark-driven overnight improvement loop.
- [pi-autoresearch](../projects/pi-autoresearch.md) packages a lighter prompt-optimization loop.
- [GEPA](../projects/gepa.md) uses reflective evolutionary search instead of brute-force prompt sweeps.
- [ADAS](../projects/adas.md) searches for better agent structures and workflows.
- [CORAL](../projects/coral.md) explores multi-agent collaboration as an improvement surface.
