# HippoRAG

> A graph-based memory and retrieval system that tries to recover long-term associativity and multi-hop reasoning. [Source](../../raw/repos/osu-nlp-group-hipporag.md)

## What It Does

HippoRAG 2 organizes knowledge into graph structure and uses personalized PageRank-style retrieval so agents can recover related concepts across hops instead of only matching local chunks. [Source](../../raw/repos/osu-nlp-group-hipporag.md)

## Architecture

The project positions itself as a bridge between RAG and long-term memory, using a knowledge graph to support sensemaking and associative retrieval. [Source](../../raw/repos/osu-nlp-group-hipporag.md)

## Key Numbers

- 3,332 GitHub stars. [Source](../../raw/repos/osu-nlp-group-hipporag.md)

## Strengths

- Strong answer to multi-hop and relation-heavy retrieval. [Source](../../raw/repos/osu-nlp-group-hipporag.md)
- Useful conceptual bridge between classic RAG and richer memory systems. [Source](../../raw/repos/osu-nlp-group-hipporag.md)

## Limitations

- Requires a graph-building pipeline that many smaller projects may not need. [Source](../../raw/repos/osu-nlp-group-hipporag.md)

## Alternatives

- [Graphiti](graphiti.md) when temporal provenance matters more than retrieval over documents.
- [PageIndex](pageindex.md) when you prefer hierarchical document reasoning to graph memory.
