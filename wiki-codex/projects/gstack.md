# gstack

> A role-based skills suite that packages CEO, design, QA, review, deploy, benchmark, and memory workflows into one opinionated stack. [Source](../../raw/repos/garrytan-gstack.md)

## What It Does

gstack installs a large set of named skills for coding, planning, design review, QA, deployment, security, benchmarking, and learned project memory. It is less a single skill than a managed skill operating system. [Source](../../raw/repos/garrytan-gstack.md)

## Architecture

The project is organized as a set of prebuilt skills discovered automatically by agents that support the `SKILL.md` standard. Its distinct move is role decomposition: different workflows are framed as different operating personas and review passes. [Source](../../raw/repos/garrytan-gstack.md)

## Key Numbers

- 63,766 GitHub stars. [Source](../../raw/repos/garrytan-gstack.md)
- Includes explicit `/benchmark`, `/canary`, and `/learn` surfaces for performance, monitoring, and memory. [Source](../../raw/repos/garrytan-gstack.md)

## Strengths

- Excellent for teams that want a ready-made workflow vocabulary. [Source](../../raw/repos/garrytan-gstack.md)
- Treats memory and benchmarking as first-class skills, not side utilities. [Source](../../raw/repos/garrytan-gstack.md)

## Limitations

- Heavily opinionated around a specific operating style and may be too much for smaller setups. [Source](../../raw/repos/garrytan-gstack.md)
