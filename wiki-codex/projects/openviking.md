# OpenViking

> A filesystem-shaped context database that combines memory, resources, and skills with layered loading. [Source](../../raw/repos/volcengine-openviking.md)

## What It Does

OpenViking stores context as files and directories with tiered loading, automatic compression, and unified handling of memories, resources, and skills for AI agents. [Source](../../raw/repos/volcengine-openviking.md)

## Architecture

The project uses an L0/L1/L2 hierarchy with small abstracts, larger overviews, and deeper content to support staged retrieval. It explicitly frames itself as a context database rather than a document retriever. [Source](../../raw/repos/volcengine-openviking.md)

## Key Numbers

- 20,813 GitHub stars. [Source](../../raw/repos/volcengine-openviking.md)
- 43% improvement over original OpenClaw with 91% lower input token cost in one native-memory setup. [Source](../../raw/repos/volcengine-openviking.md)

## Strengths

- One of the clearest examples of progressive disclosure as a storage design. [Source](../../raw/repos/volcengine-openviking.md)
- Strong benchmark positioning on token efficiency. [Source](../../raw/repos/volcengine-openviking.md)

## Limitations

- Still presented as early-stage and open to major improvement. [Source](../../raw/repos/volcengine-openviking.md)
