# Cognee

> A hybrid knowledge system that combines graph databases, vector search, and continuous contextual learning. [Source](../../raw/repos/topoteretes-cognee.md)

## What It Does

Cognee builds dynamic contextual knowledge for agents by combining graph structure with vector retrieval instead of treating the two as mutually exclusive. [Source](../../raw/repos/topoteretes-cognee.md)

## Architecture

The repo frames Cognee as a pipeline for ingesting data, building relationships, and querying through both graph and vector mechanisms so agents can operate over evolving structured context. [Source](../../raw/repos/topoteretes-cognee.md)

## Key Numbers

- 14,899 GitHub stars. [Source](../../raw/repos/topoteretes-cognee.md)

## Strengths

- Good middle ground between pure graph and pure vector systems. [Source](../../raw/repos/topoteretes-cognee.md)
- Explicitly positions continuous learning as part of the storage layer. [Source](../../raw/repos/topoteretes-cognee.md)

## Limitations

- Less opinionated than some competitors about temporal reasoning or human-legible file workflows. [Source](../../raw/repos/topoteretes-cognee.md)

## Alternatives

- [Graphiti](graphiti.md) for stronger temporal graph semantics.
- [Napkin](napkin.md) for simpler markdown-first retrieval.
